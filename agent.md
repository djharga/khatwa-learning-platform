أنت خبير واجهات UI/UX عالي المستوى ومطوّر Frontend محترف. لديك كامل الصلاحية لإعادة تصميم وتحسين جميع صفحات المشروع (عددها 101 صفحة) مع تحسين كل الملفات المرتبطة بها، وكل ملفات الستايل، والمكوّنات، والتفاعلات، والتنقل، وإدارة الحالة.

نفّذ عملية التحسين على عدة مراحل منظمة كما هو موضح أدناه. لا تغيّر أي نصوص أو محتوى تعليمي، ركّز فقط على التحسين البصري والهيكلي وتجربة المستخدم.

المرحلة 1 — تأسيس نظام التصميم (Design System Upgrade)

طبّق التطوير التالي في ملفات الستايل:

تحسين ملف tokens موحّد يحتوي:

الألوان الأساسية والثانوية

المسافات

radiuses

الظلال shadows

typography scale

transition curves

تحديث جميع الستايلات القديمة لاستعمال الـ Tokens الجديدة فقط.

تحسين تنسيق RTL عبر بنية حسابية ثابتة في الستايل.

جعل الموقع بأكمله أكثر اتساقًا في التباين، الحواف، الظلال، حجم النصوص.

نتيجة المرحلة:
أساس تصميم قوي يمكن تطبيقه على كل صفحات المشروع.

المرحلة 2 — تحسين الهيكل العام لجميع الصفحات (Master Layout & Global Experience)

نفّذ التالي على مستوى المشروع:

تحسين Layout الرئيسي في:

navbar

footer

sidebar

dashboard wrapper

إضافة تعامل سلس مع الـ dark/light mode.

توحيد أحجام الأزرار + hover + focus states.

تحسين الرسوم المتحركة transitions لجميع العناصر، مع الالتزام بسرعة 200–260ms.

نتيجة المرحلة:
تحسين فوري لكل الصفحات دون تعديل كل صفحة يدويًا.

المرحلة 3 — تحسين صفحات الطالب (Student Dashboard — 40 صفحة تقريبًا)

نفّذ التالي:

A — الكورسات (Courses & Lessons)

تحسين بطاقات الدورات وجعلها أكثر وضوحًا وتنظيمًا.

تحسين صفحات الدروس والمواد عبر:

بطاقات المواد

Icons أفضل لأنواع الملفات

فصل بصري واضح بين المحاور

تحسين الأكوردين

تحسين صفحة المشاهدة (Video/Audio/PDF Reader).

B — الامتحانات (Exams)

تحسين عرض الأسئلة.

تحسين المؤقت، ونظام التنقل بين الأسئلة.

تحسين تجربة الإجابة والحالة النشطة.

C — الملفات (File Manager)

واجهة نظيفة لرفع/تنزيل/عرض الملفات.

تحسين شكل بطاقات الملفات وتوحيد الألوان.

نتيجة المرحلة:
Dashboard احترافي بمستوى منصات التعليم العالمية.

المرحلة 6 — تحسين صفحات API واجهات الردود (API feedback UI)

المرحلة 7 — تحسين الأداء (Performance Optimization)

تحسين الصور: تحويل إلى WebP + Lazy Loading.

تحسين الـ bundling عبر dynamic imports للمكونات الثقيلة.

فحص Lighthouse وجعل النتائج ≥ 90.

تحسين التفاعلات عبر requestAnimationFrame حيث يلزم.

roles, aria-labels, aria-expanded.

تحسين focus indicators.

دعم كامل للتنقل عبر لوحة المفاتيح.

Hover واضح وجمالي للبطاقات والأزرار.
